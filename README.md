# About This Repository
This repository was made for future reference to me and anyone interested and for learning purposes. 
It is a simple, and rather violent, clone of Flappy Bird, one of the simplest games to make when learning a new technology.

# What I Learned
This was my first videogame made with the Godot game engine, so I used it to learn the various aspects, capabilities, and best practices. 
The vast majority of the code written and nodes created were made to be as robust as possible. Some of the things this taught me are:
- File structuring
- Scene structuring
- Communication between nodes and scenes
- Global singletons
- GDScript
  - Signals
  - Exporting variables properly
  - Tool scripts (used here for drawing editor helpers)
  - Interacting with the file system
  - Creating reusable components
- Audio
- 2D physics
  - The difference between static bodies and rigidbodies
  - Overriding `Rigidbody2D._integrate_forces`
- Particle systems
- Godot's UI systems
  - Normal components but also containers
  - UI skinning
- Building and exporting
